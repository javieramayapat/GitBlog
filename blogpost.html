Comandos git de básico a intermedio

Git es un sistema de control de versiones que nos ayuda a gestionar y almacenar los cambiós de nuestros proyectos. Tiene tanto poder que se ha convertido en una red social de programadores en la cual se comparten miles de lineas de código, de esa manera se motiva el codigo open source y el trabajo colaborativo.

Nota:¡Aprender git y GitHub te trae super poderes!

Tabla de comandos aprendidos

# Comando Descripción
1 git init Empieza en mi carpeta un repositorio de git, almacena espacio en memoria para el staging y empieza el track de mis archivos.
2 git add archivo.txt Añade contenido del directorio de trabajo al área de stage
3 git commit -m "mensaje" Almacena los cambios contenidos en el stage en la base atómica de git
4 git add . Agregas todos los archivos de la carpeta que han sido modificados en lugar de introducirlos uno por uno.
5 git status Mustrará los diferentes estados de los archivos en tu directorio de trabajo y área de staging.
6 git show Muestra todos los cambios echos sobre un archivo
7 git log Muestra todos los log de commits de la base de datos de git, mostrando información de quien, cuando y a que hora fueron guardados esos cambios.
8 git restore ejecicio1.txt Me ayuda a traer de vuelta un archivo que acabo de eliminar
9 git branch nombreDeLaRama Crea una nueva rama de trabajo.
10 git branch -d nombreDeLaRama Comando que borra una rama
11 git rm --cached ejecicio1.txt Significa que eliminara el archivo de la memoria ram (staging)
12 git config --global user.name "Javier Amaya" Configura el usuario global de git
13 git config --global user.email "javieramayapat@gmail.com" Configura el correo global en las credenciales de git.
14 git diff indicadorDeCommit indicardorOtrocommit Muestra los cambios entre un archivo y otro
15 git log --stat Nos ayuda ver estadísticas de las confirmaciones de commits.
16 git checkout nombreRama Ayuda a cambiar de una rama a otra.
17 git log --oneline --graph --all Muestra una linea de los commits realizado de una forma grafica.
18 gitk comando que abre una ventana donde podemos visualizar el historial de commits de manera elegante.
19 git commit -amend -m "mensaje" este comando ayuda modificar el nombre del último comando en caso de una mala escritura
20 git reset head Ayuda a quitar los archivos guardados en el staging.
21 git merge "nombre de la rama a unir" Comando que me ayuda a agrear los cambios de una rama con otra. Nota Siempre debes colocarte en la rama principal para unir la secundaria.
22 git pull origin "nombre de la rama" Trae los últimos cambios cambios del repositorio remoto y los agreg al local. Nota: Este comando siempre debe ejecutarse antes de hacer un git push, denota una buena práctica.
23 git push origin "nombre de la rama" Comando que envia los cambios del repositorio local al remoto
24 git diff head Comando que ayuda visualizar los cambios realizados en el archivo.
25 git push origin nombre_de_la_rama Comando que envía la rama creada en local a Github
26 git clone url_del_repositorio Comando que permite clonar un reposotorio de github via https o via ssh.
27 vi nombre_del_archivo Comando que permite la modificación del archivo en la consola.
28 git rebase nombreDeLaRama Comando trae la rama entera y la trae o otra rama como si la rama que se trae no hubiese existido (sin merge); cambia la historia de la rama
29 git stash Permite guardar mis avances de trabajo en un lugar de memoria, mientras hago arreglos de ultimo momento.
30 git stash pop Permite recuperar los cambios de mi avance una vez terminados los cambios de urgencia, y se guir trabajando.
31 git stash list Permite listar los stash de cambios almacenados en memoria.
32 git stash drop Elimina los cambios guardados en el stash.
33 git stash branch nombreDeLaRama Permite enviar los cambios del stash creados en la rama principal y mandarlos como stash de la nueva rama.
